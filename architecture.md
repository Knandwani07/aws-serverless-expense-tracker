# 🏗️ Architecture Overview

<img width="1376" height="768" alt="export (1)" src="https://github.com/user-attachments/assets/f4b3a4e1-d3ba-458e-a238-a56a3ff725d5" />

This project implements a fully serverless expense tracking system on Amazon Web Services (AWS). The architecture is designed to handle requests efficiently while eliminating the need for server management.

---

### 🧭 System Overview

The application follows a client-server interaction model using managed AWS services:

• A user accesses the application through a browser-based frontend  
• Requests are sent via HTTPS to Amazon API Gateway  
• API Gateway routes requests to AWS Lambda  
• Lambda processes the request and interacts with DynamoDB  
• Data is returned to the frontend for display  

This design ensures scalability, reliability, and low operational overhead.

---

### 🔄 Request Flow

1. User interacts with the Expense Dashboard (Frontend)  
2. API Gateway receives and routes the request  
3. AWS Lambda processes the request logic  
4. DynamoDB stores or retrieves expense data  
5. CloudWatch logs execution details  
6. Response is sent back to the user  

---

### 🧱 Architecture Components

#### - AWS Lambda
Handles all backend logic, including Create, Read, Update, and Delete (CRUD) operations. It processes incoming API requests and communicates with the database.

#### - Amazon DynamoDB
A fully managed NoSQL database used to store expense records. It provides high scalability, low latency, and requires minimal configuration.

#### - Amazon API Gateway
Acts as the entry point for all client requests. It exposes RESTful endpoints and routes incoming traffic to the Lambda function.

#### - Frontend (HTML, CSS, JavaScript)
Provides the user interface for interacting with the application. It sends API requests and displays expense data in a dashboard format.

#### - Amazon CloudWatch
Captures logs generated by the Lambda function. It is used for monitoring, debugging, and performance tracking.

#### - AWS IAM (Identity and Access Management)
Manages secure access between AWS services. The Lambda execution role ensures least-privilege access to DynamoDB and other resources.

---

### 🔐 Security and Monitoring

• IAM roles enforce secure communication between services  
• CloudWatch logs provide visibility into application behavior  
• API Gateway acts as a controlled entry point to backend services  

---

### ⚡ Key Characteristics

• Fully serverless architecture (no infrastructure management)  
• Automatically scales based on demand  
• Cost-efficient pay-as-you-use model  
• High availability and fault tolerance  
• Secure service-to-service communication  

---

### 🏁 Summary

This architecture demonstrates a modern cloud-native approach using AWS managed services. By combining API Gateway, Lambda, and DynamoDB, the system delivers a scalable and efficient solution for expense tracking without the complexity of managing servers.
